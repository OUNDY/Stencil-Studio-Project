# Stencil Studio

Experience-first stencil & motif e-commerce platform. Core philosophy: **discovery before instruction** — the hero section simulates the physical stencil experience before any product is shown.

**Stack:** React 18 + TypeScript + **Vite** + Tailwind CSS + shadcn/ui + Framer Motion + React Query + React Router DOM

> **Note for AI tools:** This is a **Vite** project, not Next.js. Ignore any `"use client"` directive suggestions — they are Next.js-only and would cause syntax errors here.

---

## Architecture Map

```
src/
├── App.tsx                  # Route definitions + provider tree (entry point)
├── pages/                   # One file per route; account/ subdir for /hesabim/*
├── components/
│   ├── hero/                # StencilCanvas, SurfaceReveal, HeroExperience — canvas logic lives here
│   ├── sections/            # Landing page sections (CTASection, FeaturesSection, etc.)
│   ├── navigation/          # Navbar, FloatingCart, ChatbotWidget, GlobalWidgets
│   ├── admin/               # Admin panel sub-components
│   ├── account/             # Account layout sub-components
│   └── ui/                  # shadcn/ui primitives — DO NOT hand-edit these
├── context/
│   ├── AuthContext.tsx      # localStorage-based mock auth; useAuth() hook
│   └── CartContext.tsx      # Cart state; useCart() hook
├── hooks/                   # use-mobile, use-toast
├── lib/utils.ts             # cn() helper only
└── assets/                  # Static images (wall-empty.png, wall-painted.png, steps, features)
```

Routes use **Turkish slugs** (`/koleksiyon`, `/hesabim`, `/giris`, etc.) — keep this consistent when adding routes.

## Code Conventions

- **Imports:** Use `@/` alias for all `src/` imports — never use relative `../../` paths.
- **Styling:** Tailwind utility classes only; never inline styles. Compose with `cn()` from `@/lib/utils`.
- **New shadcn components:** Add via CLI (`npx shadcn@latest add <component>`), not by hand-writing to `src/components/ui/`.
- **Context hooks:** Always consume via `useAuth()` / `useCart()` — never import context object directly.
- **Component naming:** PascalCase files and exports. Section-level components go in `sections/`, not `pages/`.
- **New pages:** Register route in `App.tsx` above the `*` catch-all; use Turkish slug.

## Gotchas & Warnings

- **`StencilCanvas.tsx` is the most complex file** (~19KB, canvas + RAF loop + mask erase). Be surgical here — any change can break the reveal mechanic. Read the full file before editing.
- **Auth is mock/localStorage only** — `AuthContext` has no real backend. Do not add Supabase auth without replacing the entire context; partial integration will cause state conflicts.
- **`src/components/ui/`** is generated by shadcn/ui. Editing these files directly will be overwritten on the next `add` command; extend via wrapper components instead.
- **Two toaster systems exist** (`Toaster` from `ui/toaster` + `Sonner` from `ui/sonner`) — both are mounted in App.tsx. Use `sonner` (`toast()`) for new notifications; don't add a third.
- **`wall-empty.png` and `wall-painted.png`** are ~2.7MB each — don't duplicate or re-import outside `hero/`. Optimize before adding new large assets.
- **No test command** — `npm run build` is the only automated check. Type errors surface here; run it before considering a task done.

## Git & Workflow

Currently solo project. Suggested minimal convention:
- Branch: `feature/<topic>` or `fix/<topic>` off `main`
- Commits: `feat:`, `fix:`, `refactor:`, `style:` prefixes
- Deploy: Netlify auto-deploys on push to `main` — don't push broken builds
